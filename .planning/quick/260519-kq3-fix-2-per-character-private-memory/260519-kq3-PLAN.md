<objective>
Replace the full-transcript context fed to each character with a per-character private memory: visible messages (own + replyTo + mention + user) + a one-line per-character summary of everything else, placed at the top of the prompt. Pattern from Generative Agents / Smallville.

Purpose: Stop bots from fabricating content from messages they "weren't in the room for." Fix 1 verification showed Reeva/Zara riffing on "linkedin" / "dead end roles" for a Delhi-heat prompt because each bot saw the full transcript.

Output: New `private-memory.ts` module + unit tests + orchestrator wired to use it.
</objective>

The per-character summary should be prepended ABOVE the identity line `You are ${friendName} ...` — either by adding a new first parameter `summaryAtTop` to `buildPrompt`, or by string-concatenating in the caller.
